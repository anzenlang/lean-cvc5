    Cvc5 Lean 4 API #

    This library exposes the C++ API of the cvc5 SMT solver in Lean 4. While this library tries to stay as close as possible to the C++ API, some changes are required to preserve Lean 4's soundness and memory/thread safety.

    The main issue is dealing with the fact that for some types, the values created by cvc5 are only safe to use as long the "manager" that created them is alive in memory. This is the case for terms, sorts, solvers, etc. Also, none of these types are thread-safe, because their manager isn't.

    This library blocks unsafe usage of these types through a pattern resembling Lean's ST.Ref type. In practice, this means most functions in this library make use of the cvc5.Env monadic environment.

    Most important sections of this documentation:

    • Environment monad and Solver (scoped) type definitions
    • Basic scoped types, e.g. terms (Term) and sorts (Srt)
    • Srt and Term interfaces
    • Solver interface

    Cvc5 monadic environment #

    source
    structure cvc5.Env ( : Type) :
    Type

    Environment monad for most cvc5 functions, based on Lean's ST.Ref.

    Env represents code producing a value of type in scope . In C++-level terms, the "scope" corresponds to a specific term manager being alive, which means everything created by this manager is legal to use: terms, sorts, solvers, etc.

    As a consequence most types in this library take a scope type parameter that encode scope traceability as type-checking, similar to Rust's lifetimes. See for instance Term and Solver which can only be created inside an Env environment.

    For technical details on how this scoping works, see cvc5.run.

      Scoping with #

      Notice the type of the code argument is { : Type} Env , not Env with a type parameter of the run function. This means that whatever will end up being passed to code, it cannot escape its scope which is the body of this run function.

      For instance, consider a hypothetical function producesTerm : { : Type} Env (Term ). Remember Term is the type of terms that are only legal for scope , i.e. inside an Env . If we tried to run this function then the in run's signature would need to be Term , but this is impossible as cannot exist outside the run function.
